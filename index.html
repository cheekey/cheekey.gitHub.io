gulp基本用法

gulp基本用法

首先先看下gulp能做什么，能为我们带来哪些好处！
gulp是注重流程方面的前端自动化工具，gulp引入一些插件能做些什么比如（编译sass,压缩js,图片转base64，等等，前提是记得引入插件）下面简单说下gulp的API和一些常见的插件！

gulp.src()

gulp.src(globs[,options])
参数globs是文件的匹配模式，匹配文件路径，该参数可以是数组，可以同时有多个匹配模式；看例子

（’app/js/.js’）匹配文件路径为app下的js下的所有.js文件；
（’app/js/a.js’）直接指某个文件
（’app/**/.js’）匹配app下级的所有js文件以及下级中所有下级的js;有点绕简单来说app下的所有.js文件都会被匹配到；
（’!app/js/d.js’）！匹配除了app/js/d.js的所有文件
([‘app/js/*.js’,!app/js/d.js’]) 这个匹配用到了以上两种，会匹配app下的js下的所有js但是d.js除外；
例子到这里以上只举了一些来说还有其他的，灵活运用吧！

常用插件

gulp.dest(path[, options])

处理输出，并且将会写文件。并且重新输出所有数据，可以输出到多个文件夹。如果某文件夹不存在，将会自动创建它。

gulp.watch(glob[, opts], tasks)

路由引用


import {Router, Route, hashHistory} from 'react'

route

路由定义通过route的path属性定义路由
Router 是容器，有个history属性可以定义路由的方式 


ReactDom.render((
  <Router history={hashHistory}>     
    <Route path='/' component={Index}/>       
    <Route path='/home' component={Data}/>     
  </Router>      
  ),document.getElementById('content')     
)

path属性定义路由
跟路由加载Index组件，/home加载Data组件，不单单加载Data组件，会先加载index组件然后再加载Data组件

IndexRoute

指定根路由加载的组件，根路由加载一个组件这个组件本身并没有东西，只负责让子组件加载进去


import React,{Component} from 'react'

export default class App extends Component{
  constructor(props){
    super(props);
    this.state={};
  }
  render(){
    return <div>{this.props.children}</div>
  }
}

我们用IndexRoute指定根路由组件，此时的this.props.children就是我们IndexRoute指定加载的组件。

IndexRedirect根路由路由重定向


ReactDom.render((
  <Router history = {hashHistory}>
    <Route path = '/' component={App}>
      <IndexRoute component = {Index}/>
      <IndexRedirect to="/po" />
      <Route path = '/po' component = {Data}/>
    </Route>
  </Router>
  ),document.getElementById('content')

这时根路由变成加载Data组件，to的值是路由；

行元素与块元素

常见块级元素：div p form ul ol li ，常见的行内元素有：span strong em ，

区别一：

块级：块级元素会独占一行，默认情况下宽度自动填满其父元素宽度

行内：行内元素不会独占一行，相邻的行内元素会排在同一行。其宽度随内容的变化而变化。

区别二：

块级：块级元素可以设置宽高

行内：行内元素不可以设置宽高

区别三：

块级：块级元素可以设置margin，padding

行内：行内元素水平方向的margin-left; margin-right; padding-left; padding-right;可以生效。但是竖直方向的margin-bottom; margin-top; padding-top; padding-bottom;却不能生效。

区别四：

块级：display:block;

行内：display:inline;

可以通过修改display属性来切换块级元素和行内元素
